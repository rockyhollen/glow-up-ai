# Glow-Up AI — Launchable MVP Repo

A complete MVP for **Glow-Up for Men**, an AI-powered paid product that collects client photos, generates a personalized appearance optimization report, and delivers a PDF.

## What is included

- Next.js App Router frontend
- Landing page copy and UI
- Stripe Checkout route
- Photo upload + intake form
- OpenAI vision prompt engine with strict JSON output
- Personalized hair, beard, skin, DIY, clothing, styling, product, and behavior recommendations
- PDF report generation
- Optional email delivery via Resend
- Production prompt in `/lib/prompts.ts`

## Setup

```bash
npm install
cp .env.example .env.local
npm run dev
```

Open `http://localhost:3000`.

## Required services

1. OpenAI API key.
2. Stripe product and price.
3. Optional: Resend for email delivery.

This repo uses Next.js Route Handlers, which are request handlers inside the `app` directory, and Stripe Checkout Sessions for hosted payment. The implementation follows current Next.js and Stripe patterns. See cited docs in the ChatGPT handoff response.

## Launch flow

1. Customer clicks **Get My Blueprint**.
2. Stripe Checkout opens.
3. After success, customer returns to `/upload?session_id=...`.
4. Customer uploads 3–5 images and intake info.
5. `/api/generate` verifies payment when `PAYMENT_REQUIRED=true`.
6. OpenAI generates the structured report.
7. Customer sees the report and can download PDF from `/api/report-pdf`.

## Local testing

Set:

```env
PAYMENT_REQUIRED=false
```

Then go directly to `/upload`.

## Deployment

Recommended: Vercel.

Set these environment variables in Vercel:

```env
OPENAI_API_KEY=
OPENAI_MODEL=gpt-4.1
NEXT_PUBLIC_APP_URL=https://yourdomain.com
STRIPE_SECRET_KEY=
STRIPE_PRICE_ID=
PAYMENT_REQUIRED=true
RESEND_API_KEY=
FROM_EMAIL=
```

## Stripe

Create a one-time product in Stripe such as:

- Product: Glow-Up Blueprint
- Price: $29
- Copy the Price ID into `STRIPE_PRICE_ID`.

## Notes

- The app sends uploaded images as base64 data URLs to OpenAI. For larger scale, move image storage to Cloudinary or S3 and pass signed URLs.
- For deterministic formatting, the prompt forces strict JSON and the app validates basic shape before rendering.
- Product recommendations are generated by the model. For live product search, connect affiliate APIs or shopping APIs to the `products` fields after report generation.


## UI / Landing Page Build

This repo now includes a polished customer-facing UI:

- `/` — premium landing page with hero, benefits, sample output, pricing, and CTA
- `/upload` — client intake flow with goal, budget, maintenance level, style preference, photo previews, loading state, and report rendering
- `components/ReportView.tsx` — full premium report UI with sections for:
  - celebrity direction
  - face diagnostic
  - hair system and barber script
  - beard system
  - skin/self-care
  - style system
  - presence and execution plan

### Brand direction

The UI is intentionally dark, premium, masculine, and minimal:
- black / warm gold / soft neutrals
- large editorial typography
- card-based diagnostic layout
- PDF-friendly report structure
